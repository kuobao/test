Place a New Order

Place a New Order

This endpoints places a new order from user's cash account.

POST api/v2/cash/order

Requested Parameters

FieldName FieldType Required Default Description Value Range Example
coid string Yes NA a unique identifier of length 32 "xxx .... xxx"
time long Yes NA milliseconds since UNIX epoch in UTC 1528988100000
symbol string Yes NA "ETH/BTC"
orderPrice string No NA required for limit orders and stop orders "13.5"
stopPrice string No NA required for limit orders and stop orders "15.7"
orderQty string Yes NA "3.5"
orderType string Yes NA "limit", "market", "stop_markter", "stop_limit" "limit"
side string Yes NA "buy", "sell" "buy"
postOnly boolean No False if true, the order will either be posted to the limit order book or be cancelled, i.e. the order cannot take liquidity true
timeInForce string No "GTC" Currently, we support "GTC" (good-till-canceled) and "IOC" (immediate-or-cancel). "GTC, "IOC" "GTC"

Response Content:

FieldName FiledType Description
coid string
action string
success boolean 'true' means the order has been submitted to the matching engine

Successful Response Example:

{
    "status": 200,
    "timestamp": 1499223904680,
    "data":{
        "coid": "xxx...xxx"
        "action": "new"
        "suceess": true
    }
}
