Cancel an Order

Cancel an Order

This endpoint submits a request to cancel a cash account open order

DELETE api/v2/cash/order

API Key Permission: Trade

Requested Parameters

FieldName FieldType Required Default Description Value Range
coid string Yes NA a unique identifier of length 32 which consists of only lower case characters (a-z), upper case characters (A-Z) and digits (0-9)
origCoid string Yes NA the coid of the order to cancel
time long Yes NA milliseconds since UNIX epoch in UTC, requests placed more than 30 seconds ago are treated as expired and will not be processed
symbol string Yes NA Trading symbol to trade
sync boolean No False if true, the 'status' field in response will demonstrate the status of the order in matching engine (note that it will slow down the response speed) True, False

Response Content:

FieldName FiledType Description
coid string a unique identifier of length 32 (see above)
status string request status (see below)

Request body: application/json

Example:

{
    'coid': "lDk7dmxzPMc28ysU76ZfWlT7egaj5Vl4",
    'origCoid': "BqkMET6srJ7iou2p08wQUGKTf5dCtSi0",
    'time': 1561043163688,
    'symbol': "BTH/ETC", 
    'sync': None
}

Successful Response Example:

{
   'code': 0,
   'data': {'coid': 'lDk7dmxzPMc28ysU76ZfWlT7egaj5Vl4', 'status': 'submitted'},
   'email': 'kuo.bao@test.com',
   'timestamp': 1561043164195
}

Other Info:

  • For this API you must include x-auth-coid in your request header.
  • Note that you must use the cash account API to cancel cash orders and margin account API to cancel margin orders.
  • Request status: The status of a request to cancel a cash account open order can be 'Rejected', 'Canceled' or 'Submitted'. 'Rejected' means the request has been rejected and 'Canceled' means the cash account open order has been successfully canceled. If the status is 'submitted', it means that the request has been received by the server. However, it doesn't imply that the order has been successfully canceled in matching engine. If 'sync' field in request body is false, the 'status' field in response will always be 'submitted'. API users who want to know the status of the request should set 'sync' filed in the request body to be true (note that it will slow down the response speed and the 'status' field in response can still be 'submitted'). API users are highly recommended to use websocket to monitor the status of the order placed.
  • Note that to allow clients to cancel immediately after placing an order, we do not check against open oders when user submits a request to cancel a cash account open order. It means that if there is no order in the matching engine whose coid is the origCoid in the request body, the response will still be a successful response instead of a failed resposne with error code and message.
